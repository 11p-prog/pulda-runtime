# ADR-0015 — Persistent Runtime Database on Autoscale

Status: Proposed — implementation prepared, deployment evidence pending  
Decision date: 2026-07-19  
Canonical authority: Notion CR-0015 persistence hold and user-confirmed Replit Autoscale

## Context

Pulda Runtime is deployed with Replit Autoscale. A deployed SQLite Event was no
longer present after redeployment (`Event ID 1` before deploy, `events: 0`
after deploy). File-local SQLite therefore cannot serve as the authoritative
Runtime store across redeployments or multiple Autoscale instances.

## Decision

- Keep Replit Autoscale for the MVP execution environment.
- Use PostgreSQL whenever `DATABASE_URL` is configured.
- Keep SQLite as the local-development and portable fallback.
- Keep service and API behavior independent from the selected database.
- Serialize PostgreSQL schema initialization with an advisory transaction lock
  so concurrent Autoscale startup is safe.
- Do not claim persistence until one Event survives a redeployment check.

## Consequences

- Replit credentials remain environment-managed and are never committed.
- PostgreSQL becomes the deployed operational evidence store.
- SQLite remains useful for local tests and offline export, but not as the
  deployed Autoscale source of truth.
- Backup, JSON export, and a future host migration remain required operational
  capabilities; this ADR does not bind Pulda Runtime permanently to Replit.

## Verification

1. Deployment health reports `database_backend: postgresql`.
2. The complete automated suite passes on SQLite.
3. A privacy-reviewed Daily Activity Event is created in deployed PostgreSQL.
4. Re-deploying the same commit preserves the Event ID and item count.
5. Replaying the Notion envelope adds no duplicate Event or item.
