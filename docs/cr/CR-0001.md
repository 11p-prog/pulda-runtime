# CR-0001 — Workspace UX

## Background
Pulda's first UI pass (Stitch design) read as a traditional dashboard/todo
app: a "Good Morning" greeting hero, a checklist-style task list, and a
sidebar-only capture field.

## Problem
- Pulda is not a to-do app; it operates Context Workspaces (domain ×
  time), and the UI did not communicate that.
- Capture was tucked into a sidebar menu instead of feeling like the
  primary interaction ("입력은 메뉴가 아니라 호흡").
- No way to move between domain contexts (Business/Family/Church/
  Personal/Knowledge) without losing the current working state.

## Goals
- Make Context Workspace the organizing structure of the whole screen.
- Make Event capture feel like a constant, low-friction action, not a
  buried form.

## Requirements
- Center content organized into workspace tabs (Today, one tab per active
  project, Church, Family, Knowledge placeholder), VS Code/Replit-style.
- Replace the "Good Morning" hero with an operational status summary
  (focus/waiting/blocked/completed counts), not a greeting.
- Do not present Pulda as a checklist app — UI communicates the Event
  lifecycle (captured → classified → in progress → done/deferred/blocked),
  not a todo list.
- Event capture bar fixed at the bottom center (ChatGPT-input-like), always
  visible, auto-tags the active workspace's context.
- Sidebar holds navigation only (mini calendar, workspace tab list) — per
  the principle "Navigation exists to find. Workspace exists to work."

## Acceptance Criteria
- Switching workspace tabs changes the center content without a full page
  navigation feel (same page, different context).
- Capture bar is reachable without opening any menu, from any workspace.
- No literal "Good Morning" or generic greeting text remains.

## Future Considerations
- Additional domain tabs (e.g. per-client workspaces) should reuse the
  same tab mechanism, not a new UI pattern.

## Affected Components
`pulda/service.py::context_workspace`, `CONTEXT_TABS`, `pulda/app.py`
(`index()`), `pulda/templates/index.html` (tabs bar, hero, capture bar).

## Status
