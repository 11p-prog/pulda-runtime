# CR-0002 — Calendar = Date Context

## Background
The mini calendar existed but selecting a date only highlighted a cell —
it did not change what the center workspace showed.

## Problem
Date is not a filter on a dashboard; it is a second, independent context
axis alongside domain. Selecting 2026-07-10 should open "2026-07-10의
워크스페이스" — that day's actual record — not just decorate the calendar.

## Goals
Make date selection a real context switch: the center workspace, stats,
recent events, full event list, and review all reflect the selected date.

## Requirements
- `context_workspace(ctx, selected_date)` scopes plan/stats/recent/events
  to the selected date, not always "today".
- Live-only state (Waiting/Overdue/Deferred) only renders for the live
  Today view — a historical date is a record of what happened, not a
  pretend live dashboard.
- `review_for_date(review_date)` shows the review matching the selected
  date, not always "latest review".
- Calendar remains navigation (Product Principle #3) — it selects dates,
  it is not itself the primary workspace.
- A date becomes "pinned" only when explicitly clicked (`cal_date` query
  param present); the unpinned/live view continues to track real "today".

## Acceptance Criteria
- Clicking a past calendar day changes the center workspace to that day's
  actual plan/events/review.
- Live-state sections (Waiting/Overdue/Deferred) do not appear on
  historical dates.
- Clicking "Today로 돌아가기" returns to the live, unpinned view.

## Future Considerations
- Retroactively generating a review for a past date the user never
  reviewed at the time is out of scope — the current data model doesn't
  support reconstructing "what was overdue at the time" for a past day.

## Affected Components
`pulda/service.py` (`context_workspace`, `review_for_date`), `pulda/app.py`
(`index()` — `pinned`/`is_today`/`selected_date`), `pulda/templates/index.html`
(header, sidebar calendar, hero, event lifecycle card, Recent Events,
review section).

## Status
Implemented — built and verified via screenshot (live Today vs. a past
date, both directions of navigation). Awaiting User Verified.
