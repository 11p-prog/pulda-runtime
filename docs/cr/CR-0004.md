# CR-0004 — Automatic Today

## Background
During a work session that crossed midnight, the app stayed stuck showing
the previous day as "today" until a manual reload.

## Problem
"Today" must follow real time automatically, but a user who has
deliberately navigated to a historical date should not be yanked back to
Today underneath them.

## Goals
Live workspace always reflects the real current date without requiring a
manual refresh, while manual historical navigation is preserved until the
user explicitly returns.

## Requirements
- The server-rendered header exposes the server's current date and
  whether the view is pinned to a manually selected date.
- Client-side polling compares the browser's local date to the
  server-rendered date; on mismatch, reload — but only when unpinned.
- A pinned historical view (via `cal_date`) never auto-reloads.
- A visible "Today" button/link is always available to return to the live
  view from any historical date.

## Acceptance Criteria
- Leaving the app open across local midnight updates to the new day
  without a manual refresh, when viewing the live (unpinned) workspace.
- Viewing a manually selected historical date is not disrupted by the
  passage of midnight.
- A "Today" affordance is visible whenever `is_today` is false.

## Future Considerations
- Polling interval (currently 30s) can be tuned; a `visibilitychange`-based
  check-on-focus could reduce polling further if needed.

## Affected Components
`pulda/app.py` (`pinned`, `today_iso`, `is_today`), `pulda/templates/index.html`
(header `data-*` attributes, "오늘"/"Today로 돌아가기" links, rollover JS).

## Status
Implemented — verified via `data-server-today`/`data-pinned` header
attributes and manual date-boundary check. Awaiting User Verified.
