# CR-0009 — Activity-Log Time Axis for the Event List

## Background
User feedback (relayed from the Chief Architect, 2026-07-13) on the mojibake
screenshot investigation: the discomfort wasn't event volume, it was that
"전체 Event" read as an undated pile of notes rather than a log of when
things happened.

## Problem
The "전체 Event" table had no time column and no date grouping. A person
naturally recalls activity by day ("오늘", "어제", "지난주 언제쯤"), not as
a flat list — without a visible time axis the list feels like "메모가
쌓인다" (notes piling up) instead of "활동이 기록된다" (activity being
logged).

## Resolution
- Added `수집시간` (capture time) as the first column of the "전체 Event"
  table, `HH:MM` from `created_at`.
- Added a `timeutil.date_label()` helper: 오늘 / 어제 / "N일 전" (2–6 days)
  / "M월 D일" (older), matching how a person recalls a date.
- Added `service.group_events_by_date()`: buckets an already-sorted events
  list into day groups, preserving per-event order, emitted in the order
  each day first appears (so a DESC-sorted list yields newest-day-first
  groups).
- The table now renders a full-width date-label header row before each
  day's events, on both the live Today view (which spans the whole
  history) and a historical single-date view (a single group, its own
  label).
- AI clustering of same-topic events under a collapsible group (explicitly
  named a later-stage idea, not part of this CR) was intentionally left
  out — the acceptance criterion for the discomfort found in the audit was
  a time axis, not clustering.

## Acceptance criteria (verified by test)
- `date_label()` returns 오늘/어제/N일 전/M월 D일 correctly relative to a
  given "today". (`test_date_label_relative_wording`)
- `group_events_by_date()` buckets a mixed-date events list into
  date-ordered groups without reordering events within a day.
  (`test_group_events_by_date_buckets_and_preserves_order`)

## Verification standard (per CR-0008 Definition of Done)
Backed by 2 new automated tests (12 total in `tests/test_runtime.py`, up
from 10). Manually confirmed via `curl` against the running app that the
rendered HTML contains the 수집시간 column header and 오늘/어제 group
rows in the expected order.
