# CR-0013 — Restore the First Living Loop

## Purpose

Restore the first living Pulda circulation instead of producing more disconnected UI and documents.

## Human and AI

- Human: creator, messenger, and actor.
- AI: disciple, watcher, and helper.
- The user is not a tester or approval endpoint.
- AI may actively interpret, classify, link, and draft follow-ups inside reversible guardrails.

## Required loop

Existing PDF memory or live Event → AI interpretation → human correction/action → real result → AI follow-up → correction memory/rule evolution → next interpretation uses the correction.

## Acceptance criteria

1. One PDF-derived Event and one live Event complete the same full loop.
2. Original source, evidence, confidence, model, prompt, DNA version, and revisions remain visible.
3. The user can approve, edit, reject, retry, undo, and delete.
4. One-time exceptions and reusable corrections are separate.
5. A reusable correction changes the next comparable interpretation.
6. Outcomes generate follow-up proposals.
7. Notion DNA, Git implementation history, Runtime evidence, tests, and user verification are linked.
8. A connector outage never loses the Runtime Event.
9. Another Builder can reproduce and continue the work without Replit-only state.

## Tool succession

Replit is responsible for Pulda's infancy, not permanent ownership. If it stops or Pulda advances, another Builder must inherit code, data, decisions, tests, environment requirements, deployment procedure, open issues, and next actions.
