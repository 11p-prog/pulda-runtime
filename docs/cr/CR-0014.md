# CR-0014 — First Contextual Knowledge Case

## Purpose

Prove one small information-metabolism loop with the 2026-07-14 CIO Korea
article about the Korean government's `모두의 AI` program. This is a live,
single-item capture; it does not resume the blocked historical bulk import in
CR-0005.

## User decision

Pulda must turn accumulated reading material into retrievable support for real
work and life. Originals should remain in user-owned storage, metadata should
remain portable, and relevant information should appear when a related issue,
project, or environmental change is encountered. Start with one item and prove
the loop before adding Keep, Chrome Reading List, KakaoTalk, Google Docs, PDF,
or other connectors.

## First case

- Canonical source: `https://www.cio.com/article/4196592/`
- Title: `“AI도 한글처럼”…과기정통부, 전 국민 무료 AI 챗봇 지원사업 공고`
- Publisher / published: CIO Korea / 2026-07-14
- Related context: `PRJ-PULDA-OS`, sovereign AI, model portability, public AI
  agents, action safety
- Initial archive state: `reference_only`. The canonical URL is preserved, but
  no Drive/local copy is claimed until a real user-owned storage URI and, where
  available, content hash are recorded.

## Acceptance criteria

1. Capturing the article creates an ordinary Event first and links a separate
   source-metadata record without overwriting the Event original.
2. The metadata preserves canonical URL, title, publisher, published date,
   summary, relevance, tags, related contexts, storage URI/format, archive
   status, optional content hash, and extensible JSON metadata.
3. Re-capturing the same canonical URL returns the existing record and does not
   create a duplicate Event.
4. A portable SQLite query can retrieve it using both contextual text and the
   Pulda OS project association; no vector database or cloud AI is required.
5. The API can capture and retrieve the item, and automated regression tests
   cover the cycle.
6. Missing original archival is explicit (`reference_only`), not misreported
   as complete preservation.
7. Real personal source content, databases, and attachments remain excluded
   from public Git.

## Non-goals

- Bulk migration of the user's backlog
- Google Drive, Keep, Chrome, KakaoTalk, or email connectors
- Full-text extraction, embeddings, recommendations, or offline sync UI
- A claim that the article file itself has already been archived
