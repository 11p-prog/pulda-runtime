# CR-0015 — ChatGPT Daily Activity Capture

## Purpose

Turn the user's meaningful ChatGPT activity into one date-scoped Pulda Event
without requiring manual re-entry. The first operational cutoff is 22:30 KST.

## Approved behavior

- One Event per activity date and source channel.
- Append only instructions, decisions, work results, action candidates, and holds.
- Preserve minimal non-sensitive summaries and source references, not transcripts.
- Keep ambiguous items reviewable as register, record-only, exclude, or review-needed.
- Never claim Runtime registration without a returned Event ID.

## Acceptance criteria

1. Re-sending the same day and items does not create duplicate Events or items.
2. The Runtime returns the stable Event ID and whether the batch was created or updated.
3. Source coverage and access gaps remain visible.
4. Item type, project, summary, source reference, and review state remain structured.
5. Invalid item types and empty summaries are rejected before an Event is created.
6. JSON returned by the read endpoint can be used as a portable registration record.
7. A configured ingest token is required for external write/read access.
8. Runtime registration is rejected unless the sender attests that privacy
   review was completed for the minimal summaries.
9. Automated tests cover creation, append/idempotency, rejection, retrieval,
   privacy review, and token enforcement.
10. Real Runtime execution and user verification remain separate completion gates.
11. `external_key` identifies one envelope receipt; multiple envelope receipts may
    merge into the single date/channel Event without changing earlier receipts.
12. Test envelopes use `data_class=test` and never merge into the operational Event.
13. The Notion relay resumes from a durable block checkpoint and reports Event IDs,
    total `added_count`, checkpoint, and errors for every run.
14. Daily Activity scheduling works when `AUTO_REVIEW=false`.

## Implementation

- `daily_activity_batches`: date/channel Event binding, external key, coverage and gaps.
- `daily_activity_items`: typed, reviewable, idempotent daily entries.
- `POST /api/daily-activities`: create or append and return Runtime evidence.
- `GET /api/daily-activities/{activity_date}`: portable read/export for one day.
- `PULDA_DAILY_ACTIVITY_INGEST_TOKEN`: optional in local development; required
  before exposing the ingest endpoint outside a trusted local environment.
- Notion relay adapter: ChatGPT writes one privacy-reviewed JSON envelope to
  the canonical queue page; Runtime reads it during the daily cycle and calls
  the same idempotent capture service. This avoids storing the Runtime Bearer
  token in the ChatGPT scheduled-task prompt.
- `POST /integrations/notion/daily-activities/pull`: authenticated on-demand
  queue pull for deployment verification and recovery.
