# CR-0016 — Autoscale PostgreSQL Runtime Persistence

## Purpose

Restore the persistence prerequisite for CR-0015 by moving deployed Runtime
records from instance-local SQLite to the PostgreSQL database already
provisioned for the Replit project.

## Scope

- Detect `DATABASE_URL` and select PostgreSQL in deployment.
- Preserve SQLite for local development and automated tests.
- Create the existing additive schema on PostgreSQL.
- Translate current positional SQL and SQLite `INSERT OR IGNORE` behavior
  without changing service semantics.
- Preserve insert IDs, dictionary-style rows, transactions, and retry
  idempotency.
- Expose the active database backend in `/api/health`.

## Acceptance criteria

1. Local SQLite behavior and all existing tests remain green.
2. PostgreSQL schema contains all Runtime and CR-0015 tables.
3. Autoscale startup is safe when multiple instances initialize together.
4. Deployed health reports `database_backend: postgresql`.
5. A Daily Activity Event survives a Replit redeployment.
6. Reprocessing the same Notion queue envelope returns the stable Event ID and
   adds zero duplicate items.
7. No database URL, password, or connector credential is stored in Git,
   Notion, logs, or ChatGPT.

## Status

In Progress — dual-backend implementation passes locally. Git deployment,
PostgreSQL Runtime execution, redeployment persistence proof, and user
verification remain pending.
