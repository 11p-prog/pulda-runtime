# CR-0018 — User-managed Central Workspace Views

## Decision source

User Runtime experience and explicit UX decision recorded in Notion on
2026-07-20, clarifying ADR-015 and ADR-016.

## Implemented scope

- Home remains the permanent date-based Event tab.
- Users add and close purpose views: Life, Work, Community, Statistics, and
  Settings.
- The initial add interaction is a descriptive card picker, not a compact
  dropdown that assumes prior knowledge.
- Open tabs and their order are stored independently of Event data. Closing a
  tab never deletes its underlying data.
- The implementation does not restore CR-0011's Event/Task/Goal/Project
  data-model tabs.

## Acceptance evidence

Automated tests cover permanent Home, add/remove persistence, purpose-view
discovery, and the absence of the superseded data-model tab concept. Runtime
responsive and user verification remain required.
