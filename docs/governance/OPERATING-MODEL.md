# Pulda Operating Model

## Sources of truth
- Notion: philosophy, theory, charter, architecture, domain model, decisions, reviews.
- GitHub: versioned implementation documents and source code.
- Replit: runtime implementation, deployment, testing, and rapid iteration.

## Roles
- User / Product Owner: final decisions, priorities, acceptance, real-use feedback.
- ChatGPT / PMO & Architect: consolidate decisions, preserve continuity, produce implementation-ready documents, audit gaps.
- Replit / Builder: implement approved CRs, run tests, report code-level issues.
- GitHub: code and document history, reviewable changes, release baseline.
- Notion: human-readable canonical knowledge repository and operating dashboard.

## Non-negotiable rules
1. Existing canonical documents are checked before creating new structures.
2. No duplicate theory, architecture, or governance document without an explicit supersession note.
3. UI changes require UX/IA document plus CR acceptance criteria.
4. Runtime facts override assumptions.
5. New events are captured first; hierarchy is inferred later.
