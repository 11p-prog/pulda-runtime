# UX-0004 Customer Journey

## Journey 1: First use
1. Open Home.
2. See one capture input.
3. Enter an event.
4. Confirm it appears with captured time.
5. No domain-model decision required.

## Journey 2: Daily use
1. Capture events during work and life.
2. Review chronological feed.
3. Change only operational status when necessary.
4. Check AI Insight for repeated contexts.

## Journey 3: Repetition emerges
1. Similar events accumulate.
2. AI identifies a probable stream.
3. User reviews supporting events.
4. User approves project/stream creation or dismisses.
5. Source events remain unchanged.

## Journey 4: Review
1. Open a project or stream.
2. See summarized progress and original event history.
3. Confirm bottlenecks, delays, completions, or abandonment.
4. Feed remains the historical source.
